# iMove — 相机照片传输 Android App 需求文档

## 1. 产品概述

**产品名称**：iMove

**核心目标**：解决相机存储卡中的照片/视频传输到手机的痛点。用户通过 USB OTG 连接存储设备后，可快速将照片/视频传输到手机本地，操作简单、性能优先。

**设计风格**：现代化、轻松、简洁。

**技术要求**：
- 最低支持 Android 10（API 29）
- 开发语言：Kotlin
- UI 框架：Jetpack Compose
- 设计规范：Material Design 3
- 界面语言：中文 + English 双语

---

## 2. 设备连接与识别

### 2.1 支持的连接方式

App 运行后，当手机通过 USB OTG 连接以下任一存储设备时触发目录流程（首次连接的设备弹出目录选择器，已配置过的设备自动使用记忆的目录）：

- **相机**（MTP 协议连接）
- **读卡器**（USB 大容量存储，内含相机 SD 卡）

### 2.2 设备识别

通过**存储卷 UUID + 卷标**（Volume UUID + Label）标识已连接的设备。每个设备仅记忆一个源目录。

---

## 3. 目录配置

### 3.1 首次连接

当检测到新的存储设备（首次连接）时，弹出目录选择器，让用户选择该设备上的源目录（即存放照片/视频的文件夹，通常为相机存储卡的 `DCIM` 目录）。

若已配置的设备重新连接时，之前配置的源目录在当前存储设备上不存在（如更换了 SD 卡），则弹出目录选择器让用户重新选择。

### 3.2 配置记忆

- 用户选择的目录与设备标识绑定，持久化存储在本地
- 下次连接同一设备时，自动使用上次配置的目录，无需重新选择
- 用户可在设置菜单中随时重新选择该设备的默认源目录

### 3.3 目标目录

- 传输的目标路径（手机端存放照片的位置）可在设置中自定义配置
- 默认建议路径：`Pictures/iMove/`

---

## 4. 传输模式

### 4.1 快捷传输

目录确认后，向用户提供以下快捷传输选项：

| 选项 | 说明 |
|------|------|
| 传输今日 | 传输源目录中今天的照片/视频 |
| 传输近三日 | 传输源目录中最近三天的照片/视频 |
| 传输全部 | 传输源目录中的所有照片/视频（排在最后，避免误触） |

**日期判断规则**：
1. 优先读取文件 EXIF 中的拍摄日期
2. 若 EXIF 信息不可用，回退到文件系统修改日期

**交互流程**：
1. 用户点击快捷选项
2. 弹出确认弹窗，显示符合条件的文件数量（如"检测到 23 张今日照片，确认传输？"）
3. 用户确认后，文件加入传输队列

**"传输全部"二次确认**：
- 弹窗内容示例："共 328 项文件，确认传输全部？"
- 用户点击"确认"后，文件加入传输队列

### 4.2 自定义传输

选择"自定义传输"后，进入图片/视频列表页面，用户可自行选择要传输的文件。

**列表展示规则**：
- 仅显示已配置源目录下的文件（不递归子目录）
- 支持的文件类型：照片（JPG、PNG、RAW 等）+ 视频
- 排序方式：按创建日期倒序排列（最新的在最上方）
- 布局：网格布局
- 视频缩略图：显示视频第一帧，右下角显示时长（如 02:35）。若获取时长严重影响性能则不显示
- 已传输的文件：缩略图上显示已传输标记（如右下角 ✓ 图标）
- 在传输队列中的文件：缩略图上显示队列标记（如右下角时钟/加载图标）

**网格列数**：
- 默认每行 3 张
- 用户可选择每行 1、2、3、4 张
- 用户的选择记忆在本地，下次直接按设置的布局显示

**交互模式**：

自定义传输页面有两种模式：单选模式（默认）和多选模式。

**单选模式（默认）**：
- **点击缩略图** → 将该文件加入传输队列（已在队列中的文件，点击无响应）
- **长按缩略图** → 全屏预览大图（支持左右滑动切换）
- 顶部 AppBar 右侧显示多选模式切换按钮（复选框图标 ☑）

**多选模式**（点击 ☑ 按钮进入）：
- **点击缩略图** → 选中/取消选中
- **拖动选择** → 手指滑过多个缩略图可批量选中
- **长按缩略图** → 全屏预览大图
- 顶部 AppBar 变为多选工具栏：
  - 左侧：关闭按钮（退出多选模式，返回单选模式）
  - 右侧："Move (已选数量)" 按钮，点击将选中文件批量加入传输队列
- 选中数量无上限

**全屏预览页面**：
- 全屏显示图片/视频
- 支持左右滑动切换
- 工具栏显示 "Move" 按钮，点击将当前文件加入传输队列
- 顶部：返回按钮

**性能要求**：
- 图片列表渲染需高性能，滚动流畅
- 缩略图加载采用懒加载策略，仅加载可见区域的缩略图
- 使用高效的图片加载库（如 Coil）

## 5. 传输行为

### 5.0 传输队列

所有传输操作（单选、多选、快捷传输）均通过传输队列管理：

- 文件被加入队列后，按先进先出顺序逐个传输（单线程）
- 用户快速连续点击多个文件时，文件依次入队，不会因读写速度限制而丢失
- 传输过程中，用户可以继续浏览列表并添加更多文件到队列
- 用户可在通知栏中取消整个传输队列
- 队列仅存在于内存中，App 被杀掉后队列丢失，不持久化

**队列可见性**：
- 通知栏实时显示队列状态（正在传输 X/Y，队列等待 N 个）
- 自定义传输页面可查看当前队列中的文件列表

**队列管理**：
- 队列中的文件可以单独移除（点击文件旁的 × 按钮）
- 正在传输中的文件不可移除（需先取消整个队列）
- 设备断开连接时，队列中尚未传输的文件标记为失败，已传输的文件不受影响

**缩略图状态标记**：
- 已传输完成：缩略图右下角显示 ✓ 标记
- 在传输队列中（等待/传输中）：缩略图右下角显示时钟或加载图标

### 5.1 文件存放

- 队列中的文件逐个复制到手机目标目录下
- 所有文件平铺存放，不保留原始目录结构
- 文件名保持原始文件名不变

### 5.2 重名处理

- 目标目录中已存在同名文件时，**直接跳过**，不覆盖

### 5.3 传输进度

- 使用 Android 前台服务（Foreground Service）执行传输任务
- 在系统通知栏中显示传输进度（进度条 + 已传输数量/队列总数）
- 用户可在通知中取消传输

### 5.4 错误处理

- 传输过程中若发生异常（连接断开、存储空间不足等），中断传输
- 发送通知到通知栏，写明中断原因

### 5.5 传输完成

- 传输完成后，在通知栏发送汇总通知（成功 X 个，跳过 Y 个，失败 Z 个）

## 6. 页面结构

### 6.1 首页（无设备连接状态）

当没有检测到已连接的存储设备时，显示空白页面 + 连接图标 + 提示文案：

> 请连接存储设备

### 6.2 首页（设备已连接状态）

当检测到已连接的存储设备后，以卡片布局展示传输模式选择（按优先级从上到下）：
- 传输今日
- 传输近三日
- 自定义传输
- 传输全部（放在最后，降低误触风险）

卡片风格：现代化、圆角、轻量阴影，简洁配色。右上角或菜单中提供设置入口。

### 6.3 设置页面

- 目标目录配置（手机端存放路径）
- 重新选择当前设备的源目录
- 语言切换（中文 / English）

### 6.4 自定义传输页面

**单选模式 AppBar**：
- 左侧：返回按钮
- 右侧：多选模式切换按钮（复选框图标 ☑）
- 中部：文件总数统计

**多选模式 AppBar**：
- 左侧：关闭按钮（退出多选模式）
- 右侧："Move (已选数量)" 批量传输按钮

**页面主体**：
- 可滚动的网格图片列表
- 网格列数按用户设置（1/2/3/4 列）

**队列面板**：
- 页面底部提供队列入口按钮，显示当前队列数量（如 "队列 (5)"）
- 点击展开队列面板（Bottom Sheet），列出队列中的文件
- 每个文件旁显示 × 按钮，点击从队列中移除（正在传输中的文件不显示 ×）

### 6.5 全屏预览页面

- 全屏显示图片/视频
- 支持左右滑动切换
- 图片：直接显示
- 视频：显示播放按钮，点击播放
- 工具栏："Move" 按钮，点击将当前文件加入传输队列
- 顶部：返回按钮

---

## 7. 非功能性需求

| 项目 | 要求 |
|------|------|
| 性能 | 图片列表滚动流畅，缩略图懒加载，大图预览无卡顿 |
| 权限 | USB 设备访问权限、存储读写权限（SAF） |
| 兼容性 | 最低 Android 10（API 29），适配主流机型 |
| 稳定性 | 传输过程中 App 切后台不中断，系统不杀进程 |
| 国际化 | 支持中文和英文，跟随系统语言或手动切换 |
